# 06 — 项目主页：部署到互联网

> 🕐 预计用时：25 分钟

你的插件已经在 GitHub 上了，别人可以下载安装。现在给它做一个**网页版的产品介绍页**——就像正式产品的官网一样，部署到互联网上，任何人都能通过一个链接访问。

---

## 什么是 GitHub Pages？

GitHub 提供了一个免费服务：你把 HTML 文件放在仓库里，它就帮你托管成一个网站。

- 💰 **完全免费**
- 🌐 **有独立网址**：`https://你的用户名.github.io/share-card/`
- 🔄 **推送即更新**：改了代码 push 上去，网站自动更新

---

## Step 1：创建介绍页

在 Claude Code 里输入：

```
在项目根目录下创建 docs/ 文件夹，里面做一个插件介绍网页：

1. docs/index.html — 单页面介绍站，包含：
   a. 顶部 Hero 区域：
      - 大标题 "✂️ 划词卡片"
      - 副标题 "选中文字，一键生成精美分享卡片"
      - 一个 "⬇️ 下载安装" 按钮（链接到 GitHub 仓库的 releases 或 zip 下载）
      
   b. 功能介绍区域（配 emoji 图标）：
      - ✨ 一键生成 — 选中文字，右键即可生成卡片
      - 🎨 自定义风格 — 多种卡片模板可选
      - 📋 自动复制 — 卡片自动复制到剪贴板
      - 📱 扫码溯源 — 二维码直达原文

   c. 截图展示区域：
      - 放 2-3 张示意图的占位框（后续可以替换为真实截图）
      - 标注 "使用效果展示"

   d. 安装步骤区域：
      - 第 1 步：下载插件文件
      - 第 2 步：打开 chrome://extensions/
      - 第 3 步：开启开发者模式
      - 第 4 步：加载已解压的扩展程序

   e. 页脚：
      - "Made with ❤️ and Vibe Coding"
      - GitHub 仓库链接

2. docs/style.css — 样式文件：
   - 现代、简洁、有质感
   - 响应式布局（手机也好看）
   - 适当的动画（滚动渐入效果之类的）
   - 配色和你给插件设计的风格统一

3. 整体参考 Apple 产品页面的简洁大气风格
```

## Step 3：部署到 GitHub Pages

你的仓库已经在 GitHub 上了，只需要开启 Pages 功能：

1. 在 GitHub 仓库页面，点击 **Settings**（设置）
2. 左侧找到 **Pages**
3. Source 选择 **Deploy from a branch**
4. Branch 选择 `main`，文件夹选择 `/docs`
5. 点击 **Save**

然后把 docs 文件夹推送上去：

```
帮我提交并推送到 GitHub
```

等待 1-2 分钟，GitHub 会自动部署。

> **Vibe Coding 的本质不是"不写代码"，而是"把创造力从语法中解放出来"。**
