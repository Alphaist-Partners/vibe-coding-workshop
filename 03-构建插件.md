# 03 — 构建插件：划词卡片

> 🕐 预计用时：40 分钟

这是工作坊最核心的章节。我们通过 4 步对话，让 Claude Code 搭建完整的划词卡片插件。

---

## 整体规划

```
划词卡片插件
├── ✂️ 文字捕获 — 用户在网页上选中文字，右键触发
├── 🎨 卡片渲染 — Canvas 绘制：高亮 + 暗化 + 来源 + 二维码
├── 📋 复制到剪贴板 — 自动把卡片图片写入剪贴板
└── 💾 历史记录 — 保存生成过的卡片
```

---

## Step 1：右键菜单 + Content Script

让插件能"感知"用户在网页上选中了什么。

在 Claude Code 里输入：

```
给划词卡片插件添加核心交互功能：

1. 添加右键菜单（Context Menu）：
   - 用户选中网页文字后，右键出现 "✂️ 生成分享卡片"
   - 需要在 manifest.json 声明 contextMenus 和 activeTab 权限
   - 创建 background.js 作为 service worker 处理右键菜单

2. 创建 content script（content.js + content.css）：
   - 当用户点击右键菜单后，content script 被激活
   - 获取用户选中的文字（window.getSelection）
   - 同时获取选中文字的上下文（前后各取一段文字，大约各 100 字）
   - 获取当前页面的标题（document.title）和 URL（location.href）
   - 先用 console.log 把这些信息打印出来，确认能拿到

3. 在 manifest.json 里配置好 content script 和 background script
```

### 验证

1. 刷新插件
2. 打开一个**普通网页**（比如百度新闻、知乎文章、任意博客）
3. 选中一段文字，右键，应该能看到 "✂️ 生成分享卡片"
4. 点击后，打开浏览器开发者工具（F12）的 Console，应该能看到打印的信息

> ⚠️ **注意**：不要在 Chrome 内部页面（`chrome://extensions/`、新标签页、Chrome 商店）上测试——这些页面不允许插件注入代码。如果右键菜单没出现，先换个普通网页试试。

> 💡 **Content Script 是什么？** 它是插件注入到网页里运行的代码。它能"看到"网页内容，但和网页本身的代码互不干扰。

## Step 2：Canvas 绘制卡片

这是最关键的一步——把捕获到的信息"画"成一张好看的卡片。

```
在 content.js 里添加 Canvas 卡片绘制功能：

1. 创建一个 Canvas 元素（不需要显示在页面上，在内存里操作）
   - 卡片尺寸：宽 720px，高度根据内容自适应

2. 绘制卡片内容（从上到下）：
   a. 背景：纯白色（后续可以换）
   b. 上方留一些 padding
   c. 上下文文字：浅灰色（比如 #999），14px 字号
      - 如果有上文，显示上文（可能需要截断，末尾加 "..."）
   d. 高亮选中文字：深黑色（#1a1a1a），20px 字号，加粗
      - 左边画一条竖线（accent color，比如蓝色或橙色）作为引用标记
   e. 如果有下文，显示下文，同样浅灰色
   f. 分隔线
   g. 底部信息栏（左右布局）：
      - 左边：网站标题（14px 深灰色）+ 换行 + "长按识别二维码查看原文"（12px 浅灰）
      - 右边：文章 URL 生成的二维码（80x80）

3. 二维码使用免费 API 生成：
   https://api.qrserver.com/v1/create-qr-code/?data={URL}&size=80x80
   加载为 Image 后画到 Canvas 上

4. 文字绘制需要处理自动换行（Canvas 不会自动换行，需要手动计算）
```

### 验证

选中文字 → 右键 → 生成卡片。打开 Console 看看有没有报错。此时卡片画好了但还看不到（在内存里），下一步就让它显示出来。

## Step 3：预览浮层 + 复制到剪贴板

让用户看到卡片，并自动复制。

```
添加卡片预览和剪贴板功能：

1. 卡片生成后，在页面右下角显示一个悬浮预览：
   - 半透明暗色遮罩覆盖全页面
   - 中间显示生成的卡片图片（带阴影和圆角）
   - 卡片上方显示 "✅ 已复制到剪贴板"（带淡入动画）
   - 卡片下方两个按钮：
     a. "📋 再复制一次"
     b. "✕ 关闭"
   - 点击遮罩空白区域也可以关闭

2. 自动复制到剪贴板：
   - 用 canvas.toBlob() 转为图片 Blob
   - 用 navigator.clipboard.write() 写入剪贴板
   - 需要在 manifest.json 声明 clipboardWrite 权限

3. 加一个生成中的 loading 状态：
   - 点击右键菜单后，先显示一个小的 loading 提示 "⏳ 正在生成卡片..."
   - 等 Canvas 绘制完成 + 二维码加载完后切换到预览

4. content.css 里写好所有浮层样式，要好看
```

### 验证

这是见证奇迹的时刻 ✨

1. 打开任意网页
2. 选中一段你喜欢的文字
3. 右键 → "✂️ 生成分享卡片"
4. 看到 loading → 卡片预览弹出 → 提示"已复制"
5. 打开微信/飞书/备忘录，Ctrl+V（Cmd+V）粘贴——卡片出现了！

## Step 4：历史记录

把生成过的卡片存下来，方便回看。

```
给插件添加历史记录功能：

1. 在 manifest.json 声明 storage 权限

2. 每次生成卡片后，把以下信息保存到 chrome.storage.local：
   {
     id: 时间戳,
     text: 选中的文字（截取前 100 字）,
     pageTitle: 网页标题,
     pageUrl: 网页链接,
     imageDataUrl: 卡片的 base64 图片（用 canvas.toDataURL），
     createdAt: 创建时间
   }

3. 改造 popup.html 为历史记录查看器：
   - 顶部：插件名称 + 统计 "共生成 X 张卡片"
   - 下面用列表展示历史记录，每条显示：
     a. 卡片缩略图
     b. 选中的文字（截断 2 行）
     c. 来源网站标题
     d. 生成时间
   - 点击某条记录可以重新复制到剪贴板
   - 每条有 🗑️ 删除按钮
   - 如果没有历史，显示空状态提示
```

### 验证

1. 生成几张卡片
2. 点击插件图标打开弹窗
3. 应该能看到历史记录
4. 关闭弹窗重新打开，记录还在

## 遇到问题？

Chrome 插件的错误在三个地方看：

1. **网页 Console**：F12 → Console（content script 的错误在这里）
2. **弹窗 Console**：右键点击弹窗 → 检查
3. **插件后台**：`chrome://extensions/` → 插件卡片的 "Service Worker" 或 "错误"

直接把错误贴给 Claude Code：

```
console 里报错了：[粘贴错误信息]
帮我修一下
```

## 📝 本章要点

- 划词卡片的核心流程：选中 → 右键 → Canvas 绘图 → 复制到剪贴板

- Canvas API 可以用代码"画"出一张图片
