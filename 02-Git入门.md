## Git 是什么？

想象你在写一份重要文档：

```
报告_v1.docx
报告_v2.docx
报告_v2_修改版.docx
报告_最终版.docx
报告_最终版_真的最终版.docx   ← 😅
```

**Git 就是程序员的"智能存档系统"**：

- 📸 记录每次修改（谁改了什么，什么时候）
- ⏪ 随时回到任何历史版本
- 🤝 多人协作不冲突
- ☁️ 同步到云端（GitHub），不怕电脑坏了

---

## 核心概念（只需要 4 个）

| 概念 | 类比 | 说明 |
|------|------|------|
| **仓库 (Repository)** | 项目文件夹 | 被 Git 管理的文件夹 |
| **暂存 (Stage)** | 购物车 | 选好要"存档"的文件 |
| **提交 (Commit)** | 存档/快照 | 确认保存一次记录 |
| **推送 (Push)** | 上传到云盘 | 同步到 GitHub |

流程：**修改文件 → 暂存 → 提交 → 推送**

## 小步提交的好习惯

每完成一个小功能就"存档"，就像游戏里打完一关就存盘。

提交信息常用前缀：

| 前缀 | 含义 | 例子 |
|------|------|------|
| `feat:` | 新功能 | `feat: 添加右键菜单` |
| `fix:` | 修 bug | `fix: 修复截图空白问题` |
| `style:` | 改样式 | `style: 调整卡片配色` |
| `docs:` | 改文档 | `docs: 添加 README` |

## 用自然语言操作 Git

你不需要记命令，直接跟 Claude Code 说：

```
帮我看看有哪些文件被修改了        → git status
帮我提交，信息你根据改动自动写     → git add + commit
帮我查看最近 5 次提交             → git log --oneline -5
帮我撤销上一次的修改              → git revert / checkout
```
