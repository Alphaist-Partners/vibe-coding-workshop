# 05 — 发布分享

> 🕐 预计用时：15 分钟

你的插件已经能用了，也打磨了自己的风格。是时候把它推到 GitHub，让别人也能看到、用上。

---

## Step 1：推送到 GitHub

### 1.1 创建 GitHub 仓库

1. 打开 [github.com](https://github.com)，登录
2. 右上角点 **+** → **New repository**
3. Repository name 填 `share-card`
4. 选 **Public**（让别人看得到）
5. 其他保持默认，点 **Create repository**
6. 创建后你会看到一个页面——**复制 HTTPS 地址**（长这样：`https://github.com/你的用户名/share-card.git`）

### 1.2 推送代码

让 Claude Code 处理：

```
帮我把代码推送到 GitHub，仓库地址是 https://github.com/你的用户名/share-card.git
```

> ⚠️ **第一次推送需要认证身份**。推荐用 GitHub CLI（最简单）：
>
> ```bash
> # macOS
> brew install gh
> # Windows
> winget install --id GitHub.cli
> ```
>
> 安装后运行：
>
> ```bash
> gh auth login
> ```
>
> 按提示选择：**GitHub.com → HTTPS → 用浏览器登录** → 完成。
>
> 如果遇到问题，举手让教练帮你。

推送成功后，刷新 GitHub 仓库页面——你的代码出现在了互联网上！🎉

```
帮我提交，信息写 "feat: 推送到 GitHub"
```

## Step 2：写一个好看的 README

现在你的仓库主页还很简陋。README 就是仓库的"门面"：

```
帮我写一个 README.md，包含：
- 项目名称和一句话介绍
- 功能特性列表（配 emoji）
- 安装方法（如何加载到 Chrome）
- 截图占位符（后续可以加真实截图）
- 技术栈
- "这是我用 Vibe Coding 做的第一个项目 🎉"

用中文写，语气轻松活泼
```

```
帮我提交并推送到 GitHub
```

刷新仓库页面，看看 README 是不是已经漂亮多了。

---

## Step 3：分享你的插件

告诉同事怎么安装你的插件：

1. 访问 GitHub 仓库：`https://github.com/你的用户名/share-card`
2. 点击绿色 **Code** → **Download ZIP**
3. 解压到任意位置
4. Chrome 打开 `chrome://extensions/`，开启开发者模式
5. 点击"加载已解压的扩展程序"，选择解压后的文件夹（注意选根目录，不要选子目录）
6. 完成！

**无需安装任何依赖**，下载就能用——这就是 Chrome 插件的好处。

---

## 📝 本章要点

- GitHub 是代码的"云端仓库"，push 就是上传
- README 是项目的门面——好的 README 让别人一眼看懂你的项目
- Chrome 插件的分享超简单：下载 → 加载 → 搞定
- 你的代码现在在互联网上了！
